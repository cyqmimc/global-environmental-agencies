## Commands

- `npm run dev` — start Vite dev server
- `npm run build` — production build to `dist/`
- `npm run preview` — preview production build locally
- `npm run fetch-data` — fetch latest World Bank data → `public/wb-data.json` (recommend quarterly)

No test runner or linter is configured.

## Architecture

Single-page React app displaying global environmental agencies with country-level environmental data. Built with Vite 8 + React 18 + Tailwind CSS v4 + Chart.js (Bar + Radar).

**Key files:**
- `src/App.jsx` — entire application in one component (~1200 lines)
- `public/countries.json` — curated agency data (48 countries, manually maintained)
- `public/wb-data.json` — World Bank indicator data (auto-generated by fetch script)
- `scripts/fetch-world-bank-data.js` — Node.js script to pull World Bank Open Data API (no deps, uses native fetch)
- `vite.config.js` — Vite config with `@tailwindcss/vite` plugin

**Two-source data flow:**
1. App fetches `countries.json` (curated) + `wb-data.json` (World Bank) in parallel
2. Merges by `isoCode` field — WB data lives under `country.wb` namespace
3. Curated fields (agency names, laws, treaties) are never overwritten by API data
4. If `wb-data.json` fails to load, app degrades gracefully using curated data only

**World Bank indicators** (in wb-data.json, keyed by ISO alpha-2):
- `forestArea`, `co2Mt`, `co2PerCapita` (computed), `renewableEnergy`, `pm25`, `protectedAreas`, `population`, `gdp`
- Each indicator has a `dataYear` tracking which year the data comes from (varies: 2018-2023)
- WB API uses mixed alpha-2/alpha-3 country IDs depending on indicator source; the fetch script handles both via a mapping table

**countries.json schema:** `countryEn`, `countryZh`, `agencyEn`, `agencyZh`, `descriptionEn`, `descriptionZh`, `website`, `flagUrl`, `isoCode`, `region`, `established`, `responsibilities`, `contact`, `treaties`, `epiScore`, `netZeroTarget`, `keyLaws`, `data: { forestCoverage, carbonEmission }`, `parisAgreement`, `montrealProtocol`, `cbd`.

**Treaty compliance architecture:** Three major treaties have dedicated structured objects per country:
- `parisAgreement` — status, ratifiedDate, ndcTargetZh/En (NDC commitments)
- `montrealProtocol` — status, ratifiedDate, kigaliAmendment (boolean), commitmentZh/En
- `cbd` — status, ratifiedDate, target30x30 (=30), commitmentZh/En; the 30×30 progress bar is computed at runtime by comparing `country.wb.protectedAreas` against the 30% target

Other treaties are listed as string arrays in `treaties` field with translation via `TREATY_LABELS` map.

**i18n:** inline `t(zh, en)` helper toggled by `language` state. Dedicated translation maps: `TREATY_LABELS` (17 treaties), `RESPONSIBILITY_LABELS` (9 tags).

**URL state:** All filter/sort/page/language state synced to URL via `URLSearchParams` for shareable links.

**Styling:** Tailwind CSS v4 utility classes in JSX. No custom CSS, no component library.
