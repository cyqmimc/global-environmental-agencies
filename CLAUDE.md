# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Commands

- `npm run dev` — start Vite dev server
- `npm run build` — production build to `dist/`
- `npm run preview` — preview production build locally
- `npm run fetch-data` — fetch latest World Bank data → `public/wb-data.json` (recommend quarterly)
- `npm run check-updates` — scan all data sources, report what may need updating
- `node scripts/split-countries.js` — re-split `countries.json` → `countries-core.json` + `countries-detail.json` (run after editing countries.json)

No test runner or linter is configured. See `DATA-MAINTENANCE.md` for data update sources and procedures.

## Architecture

Single-page React app: 80 countries, environmental agencies, World Bank data, treaty compliance tracking. Built with Vite 8 + React 18 + Tailwind CSS v4. Zero chart library — all charts are hand-drawn SVG components.

**Component structure:**
- `src/App.jsx` — layout, view toggle, dialogs. State logic extracted to hooks
- `src/hooks/useCountryData.js` — data fetching, WB merge, lazy detail loading, global averages
- `src/hooks/useFilters.js` — search (debounced), region/tag/compliance filters, sorting, pagination
- `src/WorldMap.jsx` — SVG world map with 4-metric coloring + hover/click
- `src/components/DetailDialog.jsx` — country detail popup with 3 tabs (Overview/Compliance/Data)
- `src/components/CompareDialog.jsx` — 2-3 country comparison table + bar chart
- `src/components/ClimateEquityView.jsx` — climate equity scatter plot
- `src/components/RankingsView.jsx` — sortable leaderboard table with composite scoring
- `src/components/Scorecard.jsx` — A-F grade report card per country (6 dimensions)
- `src/components/AboutPage.jsx` — data sources, methodology, licenses
- `src/components/charts/RadarChart.jsx` — lightweight SVG radar chart
- `src/components/charts/BarChart.jsx` — lightweight SVG bar chart
- `src/components/charts/ScatterChart.jsx` — lightweight SVG scatter/bubble chart
- `src/constants.js` — shared constants (TREATY_LABELS, RESPONSIBILITY_LABELS, NDC_RATING_CONFIG, URL helpers, CSV export)

**Data files (split for lazy loading):**
- `public/countries.json` — full 80-country dataset, source of truth (manually maintained)
- `public/countries-core.json` — slim version (~53KB) for initial load: basic fields + filter-relevant treaty flags
- `public/countries-detail.json` — keyed by isoCode (~137KB), loaded on-demand when detail dialog opens: descriptions, treaties, laws, full treaty objects
- `public/wb-data.json` — World Bank indicators (auto-generated by fetch script)
- `public/world-map.svg` — SVG world map (52KB, SVGO-compressed, CC BY-SA 3.0)

**Data flow:**
1. App fetches `countries-core.json` + `wb-data.json` in parallel (initial load)
2. Merges by `isoCode` — WB data under `country.wb` namespace
3. When user opens a detail dialog, `countries-detail.json` is fetched once and cached
4. Detail fields are merged into the country object on demand
5. If any fetch fails, app degrades gracefully

**Lazy loading strategy:**
- DetailDialog, CompareDialog, ClimateEquityView, AboutPage, RankingsView all use `React.lazy`
- Chart components are pure SVG (no chart.js) — zero additional JS cost
- Detail data loaded on-demand from `countries-detail.json`

**Composite scoring** (in RankingsView, reused by Scorecard):
- EPI 25% + Renewable Energy 20% + Forest 15% + Protected Areas 15% + Air Quality 15% + CO₂ Efficiency 10%
- Normalized 0-100, null values treated as 0
- Scorecard grades (A+ to F) computed by percentile ranking within the 80-country dataset

**Dual-row filters:** compliance status (NDC/carbon price/BTR/Kigali/30×30) + responsibility tags. Both can stack.

**View modes:** Cards (paginated grid) | Rankings (sortable leaderboard) | Climate Equity (scatter plot). Toggled via buttons.

**i18n:** `t(zh, en)` helper + `TREATY_LABELS` (17) + `RESPONSIBILITY_LABELS` (9) + `NDC_RATING_CONFIG` (7 levels).
